## REQ-001: 项目基础环境搭建

**模块：** 基础设施
**描述：**
搭建 Python 3.10+ 开发环境，安装所有依赖包（langchain、langgraph、playwright、pillow、requests 等），配置 API 密钥管理（.env），封装全局日志系统与统一异常捕获机制。

**验收标准：**
- Python 3.10+ 环境可正常运行
- 所有依赖包安装成功，playwright chrome 浏览器已安装
- .env 配置文件模板存在，支持大模型 API Key、绘图 API Key 配置
- 全局日志系统可输出到控制台和文件，包含时间戳、级别、模块名
- 统一异常捕获装饰器可用，异常信息自动写入日志

## REQ-002: SQLite 数据库设计与初始化

**模块：** 数据持久层
**描述：**
设计并创建 SQLite 数据库，包含 task_record（任务记录表）和 hot_material（爆款素材库）两张核心表，支持自动建表、数据写入与查询。

**验收标准：**
- task_record 表包含字段：id、keyword、create_time、title、content、tags、check_result、cover_path、status
- hot_material 表包含字段：id、keyword、ref_title、ref_content、ref_tags、like_num、crawl_time
- 程序启动时自动检测并创建表结构
- 支持基本的 CRUD 操作封装

## REQ-003: 小红书爆款爬虫工具

**模块：** 工具技能层（Skill1）
**描述：**
基于 Playwright 真实浏览器渲染，根据用户输入的赛道关键词，在小红书搜索页采集高赞公开笔记。支持自动下滑加载、点赞数过滤、去重、结构化输出。严格遵守风控约束：固定延时、单次最多10条、串行执行、每日≤5次（软限制）。

**验收标准：**
- 输入关键词后可自动打开小红书搜索页并采集笔记
- 支持双模式：免登录模式（默认）+ Cookie 登录模式（配置 Cookie 后自动启用）
- 支持配置参数：keyword（必填）、min_like（默认800）、max_note（默认10）、wait_delay（默认2000ms）
- 自动下滑3轮加载更多内容
- 逐条进入笔记详情页采集完整正文、标签、点赞数
- 自动过滤点赞数低于阈值的笔记
- 同一链接/标题自动去重，按点赞数倒序排序
- 输出标准化 JSON：ref_title、ref_content、ref_tags、like_num、crawl_url、crawl_time
- 页面加载超时自动重试2次（超时5秒），重试失败终止并记录日志
- 元素定位失败时跳过当前笔记，继续采集下一条
- 无符合条件笔记时返回空数据并终止后续流程
- 采集完成后自动写入 hot_material 表并生成 source_hot.json 备份
- 单次访问间隔≥2秒，禁止多线程并发
- 每日采集次数软限制（≤5次）：超限时警告提示，允许用户强制继续
- 运行环境：仅 Windows 本地开发机

## REQ-004: 爆款分析与原创文案生成工具

**模块：** 工具技能层（Skill2）
**描述：**
基于爬虫采集的爆款素材，通过 LLM（默认豆包，支持可切换适配层）先提炼爆款共性规律（痛点、标题范式、行文结构、高频关键词），再100%原创重构生成全新笔记文案。输出标准小红书格式：吸睛标题 + emoji 分段正文 + 5个精准标签。

**验收标准：**
- 输入参数：hot_material（爆款JSON）、keyword（赛道关键词）、word_count_range（可选，默认300-800字）
- 输出标准 JSON：new_title、content、tags
- LLM 调用通过统一适配层，默认豆包，支持配置切换为通义千问等其他模型
- 先分析爆款共性再创作，非直接改写
- 标题12-20字，带数字/痛点/情绪词/场景词
- 正文短句为主、段落分明、穿插 emoji、口语化
- 正文字数范围可配置（默认300-800字）
- 正文结构：开篇场景引入→痛点拆解→干货分享→总结种草→标签收尾
- 标签固定5个：1大赛道词 + 3精准垂类词 + 1热门流量词
- 创作阶段主动禁用广告法极限词
- LLM 调用超时重试2次（超时3秒），失败终止
- 返回格式自动校验 JSON 合法性，格式错误重新生成
- 内容为空/过短自动重试
- 生成后自动校验标题字数、正文篇幅、标签数量，不合格自动二次重写

## REQ-005: 双层合规检测工具

**模块：** 工具技能层（Skill3）
**描述：**
对 AI 生成的标题和正文进行双层安全检测。第一层：本地极限词库 + 在线接口（句易网）双保险匹配；第二层：LLM 语义风险审核（虚假宣传、医疗功效、低俗引流等）。任一层不通过则拦截任务，不自动修改文案。

**验收标准：**
- 输入参数：title、content
- 输出参数：check_status（bool）、check_msg（详细结果）
- 第一层检测：内置本地极限词库（数百个高频违禁词）作为基础兜底 + 在线接口（句易网）增强检测
- 本地词库覆盖所有绝对化词语（最、第一、顶级、根治、100%、万能等），命中返回具体违规词列表
- 在线接口不可用时降级为纯本地词库检测，日志标注降级状态
- 第二层 LLM 语义检测覆盖：虚假宣传、医疗功效、医美夸大、低俗引流、绝对化营销、敏感话术、诱导消费
- LLM 审核复用统一模型适配层（默认豆包）
- 任一层不通过，check_status 为 False，触发文案重新生成（最多重试3次）
- 不自动修改文案，仅检测拦截
- 所有违规内容自动记录日志
- LLM 审核超时重试2次，失败标记待人工复核，禁止自动放行
- 检测结果异常默认判定为风险内容

## REQ-006: AI封面生成工具

**模块：** 工具技能层（Skill4）
**描述：**
调用绘图 API（通过可切换适配层，支持通义万相/豆包绘图等）根据笔记标题生成小红书标准竖版封面底图（750×1000，3:4比例），再通过 PIL 叠加笔记标题文字，生成最终成品封面。风格简约高级，自动保存到任务目录。每次动态微调 Prompt 避免同质化。

**验收标准：**
- 输入参数：title（必填）、save_dir（必填）、style（可选，默认"简约高级自媒体风"）
- 输出参数：cover_path（本地绝对路径）、cover_status（bool）
- 绘图 API 通过统一适配层调用，支持配置切换不同绘图服务（通义万相/豆包绘图）
- 生成底图固定 750×1000 像素
- AI 生成底图后，使用 PIL 叠加笔记标题文字（需内置开源中文字体文件）
- 最终成品为 jpg 格式，画质高清、无水印、无模糊
- 风格干净简约、配色柔和、高级感
- 每次动态生成 Prompt，避免批量图片同质化
- 文件命名：标题前12字符 + 随机后缀
- 自动归档至当前任务文件夹
- 绘图 API 超时重试2次（超时5秒），失败终止并记录日志
- 图片下载失败提示用户手动生成，不影响文案素材
- 自动校验图片尺寸，不合格重新生成

## REQ-007: 本地智能归档工具

**模块：** 工具技能层（Skill5）
**描述：**
全流程素材结构化整理，输出可直接手动发布的完整笔记素材包。自动创建独立文件夹，生成4个标准文件（文案.txt、封面.jpg、source_hot.json、log.txt），并将任务数据写入 SQLite task_record 表。

**验收标准：**
- 输入参数：task_state（AgentState 全流程状态数据）
- 输出参数：folder_path、archive_status
- 根目录 /output/，子目录命名：YYYYMMDD_关键词_标题缩写
- 文案.txt 结构：标题单独一行、正文分段清晰、标签单独成行，无乱码
- 封面.jpg 为最终成品封面图
- source_hot.json 为本次参考的原始爆款素材，格式化缩进
- log.txt 为全流程日志：时间戳+节点名称+执行结果+异常信息
- 自动写入 task_record 任务表
- 文件夹创建失败时自动重试，失败切换临时目录
- 文件写入失败优先保留核心文案素材
- 数据库写入失败不影响本地文件归档
- 归档产出无需二次编辑，可直接复制发布

## REQ-008: LangGraph 智能体调度中心

**模块：** 调度层
**描述：**
基于 LangGraph 状态机架构，定义 AgentState 全局状态，编排5大工具节点的顺序执行与条件分支。实现：用户输入关键词 → 爬虫采集 → 文案生成 → 合规检测 → （通过→封面生成→归档→结束 / 不通过→重新生成文案，最多重试3次→全部失败则终止）完整流转链路。

**验收标准：**
- AgentState 定义包含：keyword、hot_material、raw_copy、title、content、tags、check_result、cover_path、finished、retry_count
- 状态流转：开始→爬虫节点→文案生成节点→合规检测节点→条件分支
- 合规通过：进入封面生成→归档→结束
- 合规失败：回到文案生成节点重新创作，最多重试3次
- 3次重试全部失败后终止任务，保留已有素材和失败日志
- 每个节点失败时自动终止流程并记录日志
- 支持单步骤失败重试机制
- 支持超时处理
- 全流程可通过命令行输入关键词一键启动

## REQ-009: 全局配置管理

**模块：** 基础设施
**描述：**
统一管理所有可配置项，包括 API 密钥、爬虫参数、文案生成参数、路径配置等，通过 .env 文件和配置类实现，支持环境隔离。LLM 和绘图 API 均通过适配层实现可切换。

**验收标准：**
- .env 文件管理敏感信息（API Key）
- 配置类/文件管理业务参数（爬虫延时、最大采集数、点赞阈值、文案字数范围等）
- LLM 模型可配置切换（默认豆包，可切换通义千问等）
- 绘图 API 可配置切换（通义万相/豆包绘图等）
- 爬虫模式可配置（免登录/Cookie登录）
- 配置项有默认值，缺失时不会崩溃
- 配置变更无需修改代码逻辑
- 入口支持双模式：命令行传参（`python main.py --keyword "xx"`）单次执行 + 无参启动进入交互循环

## REQ-010: 全流程日志与可追溯性

**模块：** 基础设施
**描述：**
每次任务执行的完整生命周期均有日志记录，包括各节点执行时间、输入输出摘要、异常信息、最终结果，支持问题排查与效果复盘。

**验收标准：**
- 每个工具节点执行前后均有日志输出
- 日志包含：时间戳、节点名称、执行状态、耗时、关键参数
- 异常信息完整记录（堆栈、错误类型、上下文）
- 单次任务日志同时写入控制台和任务文件夹 log.txt
- 日志级别可配置（DEBUG/INFO/WARNING/ERROR）
