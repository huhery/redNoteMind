## REQ-003: 小红书爆款爬虫工具

### 待确认问题

1. [Q] 爬虫是否需要登录态？免登录模式 vs Cookie 登录模式？
   [A] 两种都支持。默认免登录，配置了 Cookie 则使用 Cookie 访问。

2. [Q] 笔记正文采集方式：仅列表页信息 vs 逐条进入详情页采集完整正文？
   [A] 逐条进入详情页采集正文（B方案），文案生成需要完整爆款正文作为参考。

3. [Q] 爬虫运行环境：Windows本地 / CentOS服务器 / 混合？
   [A] 仅在本地 Windows 开发机运行爬虫（A方案）。Playwright 环境仅需在 Windows 上配置。

4. [Q] 每日采集次数限制≤5次的实现方式：软限制（警告但可强制继续） vs 硬限制（直接拒绝）？
   [A] 软限制（A方案）。超过5次时警告提示，但允许用户强制继续。

### 设计决策
- 爬虫模块需实现双模式：免登录 + Cookie 登录，通过配置文件切换
- 采集流程：搜索页列表 → 逐条点击进入详情页 → 采集正文/标签 → 返回列表继续
- 运行环境仅考虑 Windows，无需适配 Linux 无头环境
- 本地维护当日采集计数器（基于日期的本地文件/数据库记录），超限警告但不阻断

## REQ-004: 爆款分析与原创文案生成工具

### 待确认问题

1. [Q] 大模型 API 选型：豆包（Doubao）还是通义千问（Qwen）？是否需要可切换？
   [A] 使用豆包（Doubao）作为默认模型，但需要做成可切换的适配层，支持通过配置切换不同 LLM 服务。

2. [Q] 生成文案正文字数范围？
   [A] 可配置（C方案）。通过参数控制目标字数范围，默认可设 300-800 字区间。

### 设计决策
- LLM 调用层需抽象适配器模式，支持豆包/通义千问等多种后端切换
- 文案生成 Prompt 中加入字数控制参数，用户可在配置中指定目标字数范围

## REQ-005: 双层合规检测工具

### 待确认问题

1. [Q] 极限词检测方式：纯在线接口 / 本地词库+在线双保险 / 纯本地词库？
   [A] 本地词库 + 在线接口双保险（B方案）。内置常见极限词本地词库兜底，在线接口可用时补充检测。

2. [Q] 合规语义审核用同一个模型还是独立配置？
   [A] 使用豆包模型，与文案生成共用可切换适配层。

### 设计决策
- 内置一份本地极限词库（数百个高频违禁词），作为基础兜底
- 在线接口（句易网）作为增强检测，不可用时降级为纯本地词库
- LLM 语义审核复用统一的模型适配层

## REQ-006: AI封面生成工具

### 待确认问题

1. [Q] 绘图 API 选型：通义万相 vs 豆包绘图？是否可切换？
   [A] 做成可切换的适配层，通过配置切换不同绘图服务。

2. [Q] 首版封面是否需要叠加标题文字？
   [A] 首版就加文字（B方案）。AI 生成底图后，用 PIL 在封面上叠加笔记标题文字。

### 设计决策
- 绘图 API 层同样抽象适配器模式，支持通义万相/豆包绘图等多后端切换
- 封面生成流程：AI生成底图 → PIL 叠加标题文字 → 保存最终成品
- 需考虑中文字体文件打包（PIL 渲染中文需指定字体）

## REQ-008: LangGraph 智能体调度中心

### 待确认问题

1. [Q] 合规失败后的处理策略：直接终止 / 支持重试生成 / 终止但保留素材？
   [A] 支持重试生成（B方案）。合规不通过时自动重新生成文案，最多重试3次，全部失败才终止。

### 设计决策
- 状态机中合规检测节点后的条件分支增加重试逻辑
- AgentState 增加 retry_count 字段记录当前重试次数
- 重试流程：合规失败 → 回到文案生成节点重新创作 → 再次合规检测 → 最多循环3次
- 3次全部失败后终止任务，保留已有素材和失败日志

## REQ-009: 全局配置管理（补充）

### 待确认问题

1. [Q] 命令行交互方式：单次执行 / 交互式循环 / 两者都支持？
   [A] 两者都支持（C方案）。支持命令行传参直接执行，也支持无参启动进入交互模式。

### 设计决策
- 入口 main.py 支持 argparse 参数：`python main.py --keyword "护肤"` 单次执行
- 无参启动时进入交互循环菜单，支持多次输入关键词，quit 退出
